# Add `leaf --render` markdown renderer (new default)

## Goal

Add a new external markdown renderer that shells out to `leaf --render`,
registered via the existing `cheeto.utils.markdown.renderers` entry point, and
make it the hard-coded default choice for the `show-sheet` subcommand.

## Context

- Renderer architecture is in `src/cheeto/utils/markdown.py`.
- `MarkdownRenderer.renderers()` discovers renderers from the
  `cheeto.utils.markdown.renderers` entry-point group, instantiates each, and
  filters on truthiness (`__bool__`).
- `ExternalMarkdownRenderer.__bool__` already handles "binary not in `$PATH`"
  gracefully â€” unavailable renderers are filtered out before argparse sees them.
- `MdcatRenderer` at `src/cheeto/utils/markdown.py:90` is the closest analogue
  (stdin â†’ stdout, no width/mode flags needed).
- Per user: `leaf --render` reads markdown from stdin when no path is given, so
  we can pipe via `subprocess.run(..., input=src)` exactly like mdcat.
- Per user: default is a hard `"leaf"` â€” even if the binary is not installed.
  (Argparse will then reject the default when `leaf` is absent, forcing users
  without `leaf` to pass `-m <renderer>` explicitly. This is the user's
  explicit choice.)

## Changes

### 1. `src/cheeto/utils/markdown.py`

Add a new class after `MdcatRenderer` (order: keep it next to the other
external renderers). Mirror `MdcatRenderer` exactly, except:

- Docstring: `"""Render markdown using the leaf tool if available (default)."""`
- Executable default: `"leaf"`.
- Command: `["leaf", "--render"]` (no `-` argument; leaf reads stdin when no
  path is supplied).

```python
class LeafRenderer(ExternalMarkdownRenderer):
    """Render markdown using the leaf tool if available (default)."""

    def __init__(self, executable: Path | None = None):
        super().__init__(executable if executable is not None else "leaf")

    def __call__(self, src: str) -> RenderableType:
        cmd = ["leaf", "--render"]
        logger.debug(cmd)
        proc = subprocess.run(
            cmd,
            capture_output=True,
            check=True,
            encoding="utf-8",
            input=src,
        )
        return proc.stdout
```
